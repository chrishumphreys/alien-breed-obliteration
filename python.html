Python has a very good SDL interface in the form of Pygame. Pygame is quick to learn and quick to become productive. Knocking up an SDL game in Python is therefore very easy. I recommend it to anybody thinking of doing something similar.

I'd used Pygame before for a few smaller projects so it was a natural choice for building my Alien Breed clone. It had the distinct advantage of allowing me to develop on my Linux Eeepc on my daily commute to work, something other stacks would not have allowed.

Unfortunately Pygame hadn't been ported to the Wii environment and doing so posed a few problems.

Python had been ported in the form of WiiPy. This project consisted of a set of patches to allow cross compiling the Python interpreter using the DevkitPro GCC environment for the Wii. Additionally a set of (incomplete) LibOGC wrappers were provided to allow access to the essential Wii homebrew APIs. A significant number of Python modules had to be disabled in order to get Python to compile, notably anything to do with the OS as the equivalent behaviour was not available in the DevKitPro libraries.

Unfortunately the LibOGC wrappers were in an incomplete state and not really practical. However as my game would be based on SDL, and some kind folks had already ported SDL to the Wii. I did not need the LibOGC wrappers. They did however point the way to how an Python SDL wrapper could be built for the Wii - by using the Python/C glue technology Pyrex.

Python has a utility called Freeze which creates C bootstrap code for the Python interpreter for a set a Python files. This allows you to build a standard ELF executable for a Python program. This is not the same as converting Python source to C code which is much harder. The game is still running the Python interpreter but crucially it allows us to create a single ELF executable, statically linked with all the required SDL/Wii libraries, using the standard DevKitPro tool chain.
