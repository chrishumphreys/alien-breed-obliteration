Alien Breed is one of the all time great Amiga games. Top down maze game play similar to the arcade classic Gauntlet combined with themed levels and characters from the film Aliens made this a true Amiga classic.

I originally started to write a clone for this game in '93 for the Acorn Archimedes computer. That stalled pretty quickly as I didn't have access to any of the original media files.

So 17 years later when the Wii console homebrew scene came alive I realised it might be time to try again. This time however I had a large helping hand in the form of an existing, and excellent, PC clone Alien Breed Obliteration by FlashJesterPunk.

Wii development is not easy or quick. There is an excellent clone of the GCC tool chain and stdc libraries from the Devkitpro project. A number of other low level but essential libraries have also been ported to the platform or written to access the underlying Nintendo OS. However there is no IDE, debugger or simulator. Development is a time consuming round trip of edit, compile, copy to SDCard and try in the physical Wii.

Because of this, building a game from scratch for the Wii is a daunting task. Porting an existing portable game (for example written in C and SDL) is the more common approach.

To minimise the development turnaround I wanted to build and test the game predominantly on my Linux laptop. Building and testing on the Wii only when required. I also wanted to do this without resorting to C.

To achieve this I used Python and SDL. The game was written initially on my laptop using Python 2.4 and the SDL wrapper PyGame. This is a very portable stack, running happily on Linux, Windows and, as it turns out, the Wii.

Nobody had publicly written a Python/PyGame game for the Wii before so there was considerable challenges to getting that stack to work on the Wii. I will cover that in a different post.

I also had the difficulty of not having written a game of any significance before.
